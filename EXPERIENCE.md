# Итоги проекта MLOps (ИТМО 2026)

Этот файл содержит описание проделанной работы, обоснование выбранных решений и ключевые навыки, которые были отработаны в ходе выполнения финального проекта.

## 1. Что мы сделали?
В рамках проекта мы создали полный цикл жизненного цикла модели машинного обучения (ML Lifecycle). Основные компоненты:
1.  **Версионирование данных и моделей (DVC)**: Мы не храним тяжелые веса моделей или `.csv` файлы прямо в Git. Вместо этого мы используем DVC, который связывает Git с удаленным хранилищем (MinIO), гарантируя, что код всегда соответствует конкретной версии данных.
2.  **Трекинг экспериментов (MLflow)**: Все запуски обучения, гиперпараметры и метрики логировались в MLflow. Это позволяет сравнивать модели (например, Linear Regression vs RandomForest) и выбирать лучшую на основе объективных данных.
3.  **Оркестрация процессов (Airflow)**: Мы создали DAG, который автоматизирует переобучение модели. Это гарантирует воспроизводимость и возможность регулярного обновления модели при поступлении новых данных.
4.  **Сервинг (FastAPI)**: Реализовали REST API для предсказания качества вина. API включает в себя валидацию входных данных через Pydantic и эндпоинты для проверки состояния сервиса (`/healthcheck`) и получения информации о модели.
5.  **Автоматизация (CI/CD)**: Настроили GitHub Actions, который при каждом Pull Request проверяет код на ошибки оформления (линтер) и тестирует API.

## 2. Зачем всё это было нужно?
Основная цель проекта — перейти от простого «обучения модели в ноутбуке» к **промышленному подходу (MLOps)**.
- **Воспроизводимость**: Любой другой разработчик может склонировать репозиторий, выполнить `dvc pull` и получить точно такие же результаты.
- **Масштабируемость**: Использование Airflow и FastAPI позволяет развертывать решение в облаке или Kubernetes.
- **Безопасность**: Мы научились не хранить секретные ключи в открытом доступе (в GitHub/GitLab), используя переменные окружения и секреты CI/CD.

## 3. Чему мы научились? (Ключевые навыки)
Работая над этим проектом, мы освоили:
- **Инженерную культуру**: Соблюдение стандартов PEP 8, использование линтеров (flake8) для поддержания чистоты кода.
- **Работа с данными как с кодом**: Понимание того, что данные тоже должны иметь версии.
- **Тестирование**: Написание unit-тестов для API (pytest), которые проверяют надежность сервиса перед деплоем.
- **Автоматизация CI/CD**: Понимание того, как работают пайплайны и почему нельзя пушить код сразу в `main` без проверок.
- **Инфраструктурное мышление**: Работа с Docker (в рамках окружения), Airflow и внешними хранилищами.

## Итог
Этот проект — это не просто «нейронка на питоне». Это демонстрация того, как строить надежные, автоматизированные и проверяемые системы для решения реальных бизнес-задач с помощью машинного обучения.
