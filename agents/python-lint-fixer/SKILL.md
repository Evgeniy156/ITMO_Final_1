# Python Lint Fixer

Этот навык предназначен для систематического исправления распространенных ошибок линтинга в Python-проектах.

## Когда использовать
- Когда пайплайн CI/CD завершается с ошибкой на этапе `lint`.
- Когда в логах видны ошибки формата `src/api/main.py:107:1: W293 blank line contains whitespace`.
- Когда нужно выполнить рефакторинг стиля кода по стандартам PEP8.

## Общие инструкции по исправлению

### 1. W293: Blank line contains whitespace
Эта ошибка означает, что на "пустой" строке остались пробелы или табуляция.
**Как исправить:** 
- Удалите все символы из строки, чтобы она стала абсолютно пустой.
- **Linux:** `sed -i 's/[[:space:]]\+$//' <file.py>`
- **Windows (PowerShell):** `(Get-Content -Path "file.py") | ForEach-Object { $_.TrimEnd() } | Set-Content -Path "file.py"`
- **Recursive (Python):** `python tmp/fix_all_lint.py dags src tests`

### 2. E302: Expected 2 blank lines
Ожидается две пустые строки перед определениями функций или классов верхнего уровня.
**Как исправить:** Добавьте еще одну пустую строку перед `def` или `class`.

### 3. E501: Line too long
Строка превышает лимит (обычно 79 или 88 символов).
**Как исправить:** Разбейте строку на несколько, используя скобки `()` или символы переноса `\`.

### 4. W291: Trailing whitespace
Пробелы в конце строки с кодом.
**Как исправить:** Удалите лишние пробелы в конце указанной строки.

## Автоматическое исправление
Если в окружении доступны инструменты автоматического форматирования, используйте их:

```bash
# Использование autopep8 для исправления большинства ошибок PEP8
autopep8 --in-place --aggressive --aggressive <filename>.py

# Использование ruff для быстрого исправления
ruff check --fix <filename>.py
```

## Алгоритм работы при получении ошибок от пользователя:
1. Проанализируйте файл и номера строк, указанные в ошибке.
2. Откройте файл (`view_file`) в окрестностях этих строк.
3. Исправьте ошибку (используйте `replace_file_content` или `multi_replace_file_content`), убедившись, что не нарушена логика кода.
4. После исправления запустите локальный линтер (если доступен), чтобы убедиться, что ошибки исчезли.

### Пример исправления W293:
**До (строка 107 содержит 4 пробела):**
```python
106:     feature_dict = features.model_dump()
107:     ____
108:     # Маппинг имен полей
```

**После (строка 107 пустая):**
```python
106:     feature_dict = features.model_dump()
107: 
108:     # Маппинг имен полей
```
