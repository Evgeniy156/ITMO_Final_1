# План реализации финального проекта по модулю MLOps

В этом документе представлен пошаговый план разработки инфраструктуры MLOps-проекта (от версионирования и экспериментов до CI/CD). Отмечайте завершенные шаги для отслеживания прогресса.

## Этап 1. Подготовка репозитория и окружения
- [x] Ознакомиться с требованиями (файл проекта `README` или задание)
- [x] Создать публичный репозиторий на GitLab (или GitHub, если допустимо). Можно использовать за основу [ci-cd-example](https://gitlab.com/mlops-hub/ci-cd-example)
- [x] Создать и настроить ветки: `main` (для продакшена) и `develop` (для разработки)
- [x] Склонировать репозиторий локально и переключиться в ветку `develop`
- [x] Инициализировать DVC (`dvc init`) 
- [x] Настроить удаленное хранилище (MinIO S3) как `remote` для DVC
- [x] Добавить/проверить, что ключи доступа к MinIO добавлены в `.env` (и `.env` добавлен в `.gitignore`!)

## Этап 2. Версионирование данных и моделей (DVC)
- [x] Скачать предоставленный датасет о качестве вина по [ссылке](https://github.com/aniruddhachoudhury/Red-Wine-Quality/blob/master/winequality-red.csv)
- [x] Положить датасет в директорию `data/wine_quality.csv`
- [x] Добавить данные под контроль DVC: `dvc add data/wine_quality.csv`
- [x] Закоммитить сгенерированный файл `data/wine_quality.csv.dvc` и `data/.gitignore` в Git
- [ ] Загрузить данные в удалённое хранилище: `dvc push`
- [x] Убедиться, что сырые данные `.csv` не трекаются в Git

## Этап 3. Трекинг экспериментов
- [x] Создать Python-скрипт или Jupyter Notebook для обучения моделей
- [x] Настроить логирование с помощью MLflow (или ClearML / W&B)
- [x] Обучить 2 разные регрессионные модели (например, `LinearRegression` и `RandomForestRegressor`)
- [x] Провести несколько экспериментов с разными гиперпараметрами, зафиксировать `random_state`
- [x] Сравнить метрики качества обеих моделей
- [x] Задокументировать: сохранить лучшую модель в файл (например, `models/wine_quality_model.pkl`)
- [x] Добавить модель под контроль DVC: `dvc add models/wine_quality_model.pkl`
- [ ] Закоммитить `.dvc` файл модели в Git и запушить артефакт в S3 (`dvc push`)

## Этап 4. Оркестрация (Airflow)
- [x] Создать директорию `dags/`
- [x] Написать файл конфигурации (или использовать дефолтные `Variables` Airflow) для параметров
- [x] Создать DAG (например, `wine_pipeline_dag.py`), настроить расписание `schedule_interval='@daily'`
- [x] Реализовать task: **Загрузка данных** (чтение данных из DVC/локально)
- [x] Реализовать task: **Обучение модели** (с использованием выбранной выше лучшей модели и пайплайна)
- [x] Реализовать task: **Сохранение модели** (и метаданных) обратно в DVC (или имитация этого процесса, если настроено иначе)
- [x] Проверить корректность связей между тасками в DAG (Task1 >> Task2 >> Task3)

## Этап 5. Сервинг модели (FastAPI)
- [x] Создать директорию `src/api/` и основной файл `main.py`
- [x] Реализовать Data Validation класс с помощью Pydantic (`class WineFeatures(BaseModel): ...`)
- [x] Написать эндпоинт `GET /health` (`{"status": "ok"}`)

## Этап 6. Автоматизация тестирования и CI/CD

- [x] Настроить в CI/CD переменные окружения (Variables) для авторизации DVC в MinIO S3
- [x] Описать pipeline Stage 1: **Проверка кода линтером** (flake8, black или ruff)
- [x] Описать pipeline Stage 2: **Проверка доступности данных и модели в DVC** (команда `dvc pull`)

- [x] Настроить правила запуска CI: только при открытии Merge Request в ветку `main`
